# Git Rules

本文件适用于 Git 提交、分支、合并、回滚、PR 等任务。

## 基础原则

- 未经用户明确要求，不主动 commit、amend、push、创建 PR。
- 提交前必须检查工作区状态和 diff，避免提交无关文件。
- 小粒度提交，提交内容必须围绕同一目标。
- 不要回滚、覆盖或修改用户已有改动，除非用户明确要求。

## 提交信息

优先使用简洁语义化类型：

- `feat`: 新功能。
- `fix`: 修复问题。
- `chore`: 工程杂项。
- `docs`: 文档。
- `refactor`: 不改变行为的重构。
- `style`: 样式或格式。
- `test`: 测试。

## 风险操作

- 禁止未经确认执行破坏性命令。
- 禁止未经确认 force push。
- 禁止未经确认 amend。
- 回滚失败方案时，只回滚本次失败调整，不影响用户其他改动。
