# React Rules

本文件适用于 React 组件、Hooks、状态管理和 JSX 逻辑任务。使用前必须同时遵守 `core/scope.md`；涉及脚本逻辑时必须同时遵守 `core/coding.md`。

## 基础原则

- 优先使用函数组件和 Hooks。
- 状态保持最小化，不保存可以直接派生的数据。
- JSX 保持简洁，复杂判断提取为语义明确的变量或函数。
- 遵循项目已有 React 写法，不为了统一风格强行改造。

## Effects

- 避免复杂 `useEffect`。
- `useEffect` 只处理副作用和外部同步。
- 不要把纯计算、格式化、派生状态放进 `useEffect`。
- 复杂副作用必须拆出具备业务语义的函数。

## 状态与渲染

- 禁止无意义拆分状态。
- 禁止为缩短访问路径新增无价值中间变量。
- 列表渲染必须使用稳定业务 key，禁止默认使用 index。

## 注释

React 代码中的注释同样遵守 `core/coding.md` 的中文注释规则。
